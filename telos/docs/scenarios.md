# Telos 사용 시나리오

> 실사용 전제 — 이 시나리오들은 사용자가 해당 프로토콜을 의식적으로 호출(`/goal`)하는 상황을 기술합니다.
> Skill이 세션에 로드되지 않으면 충돌이 발생하지 않습니다.

**프로토콜**: Telos (`/goal`)
**결핍 → 해소**: `GoalIndeterminate → DefinedEndState`

---

### 1. "개발자 경험을 개선하고 싶다" `SW`

**사용자 맥락**: 팀 리드가 DX 개선을 원하지만, 무엇이 개선인지(빌드 시간 단축? 문서화? 온보딩? 개발 도구?), 어디까지가 범위인지, 성공을 어떻게 측정할지 전부 미정이다.

**트리거 발화**: "개발자 경험을 개선하고 싶다" (Outcome, Metric, Boundary, Priority 4개 차원 모두 미정)

**호출 예시**:
```
/goal 개발자 경험을 개선하고 싶은데, 빌드 시간/문서화/온보딩/개발 도구 중 어디에 집중할지부터 정해야 할 것 같아.
```

**결핍(전)**: 4개 차원 중 어느 것도 특정되지 않아 팀마다 "개선"의 의미가 달라 산출물이 분산된다.

**해결(후)**: 차원별 제안-수정 루프를 통해 "빌드 시간 30% 단축, 범위는 CI 파이프라인, 속도보다 안정성 우선" 같은 GoalContract를 확정한다.

**인접 프로토콜 배제**: **Hermeneia** — 의도 자체가 모호한 것이 아니라(DX 개선을 원하는 것은 명확), 목표의 구체적 정의가 부재하다.

---

### 2. "마이크로서비스로 가야 할까" `SW`

**사용자 맥락**: 모놀리스에서 마이크로서비스 전환을 검토하고 있지만, 어떤 도메인부터 분리할지, 완전 분리인지 점진적 분리인지, 인프라 투자 범위와 성공 기준이 불명확하다.

**트리거 발화**: "마이크로서비스로 가야 할까" (전환 여부 자체가 미확정)

**호출 예시**:
```
/goal 마이크로서비스 전환을 고려하고 있는데, 무엇을 달성하려는 건지부터 정의해 보고 싶어.
```

**결핍(전)**: Prothesis와 경계가 있지만, 여기서는 접근 프레임워크 분석이 아니라 "무엇을 달성하려는가" 자체가 불분명하여 Telos가 우선한다.

**해결(후)**: AI 제안과 사용자 수정을 거쳐 "배포 독립성 확보, 주문 도메인 우선, 6개월 파일럿" 같은 GoalContract를 만든다.

**인접 프로토콜 배제**: **Prothesis** — 프레임 부족이 아니라 완료 정의 자체가 비어 있는 문제다.

---

### 3. 스타트업 데이터 팀 구축 `KW`

**사용자 맥락**: 스타트업 COO가 데이터 팀을 만들고 싶지만, 어떤 역량의 팀인지, 성공 기준은 무엇인지, 자체 구축 vs. 외주인지, 분석 vs. MLOps vs. 데이터 엔지니어링 우선순위 등 4개 차원 모두가 미정이다.

**트리거 발화**: "데이터 팀을 만들고 싶은데, 어디서부터 시작해야 할지 모르겠어" (다차원 조직 목표 미정)

**호출 예시**:
```
/goal 데이터 팀을 만들고 싶은데 뭘 성공으로 봐야 할지, 어떤 역량부터 갖춰야 할지 모르겠어.
```

**결핍(전)**: 뚜렷한 의도는 있으나 구체적 end state가 부재. 기술+인사 교차 영역이라 복잡도가 높다.

**해결(후)**: Outcome/Boundary/Priority를 단계적으로 고정해 "분석 역량 우선, 3명 인하우스, 6개월 내 대시보드 운영" 같은 GoalContract를 만든다.

**인접 프로토콜 배제**: **Aitesis** — 필요한 사실이 부족한 게 아니라 무엇을 성공으로 볼지가 미정이다.

---

### 4. "유튜브 채널을 키우고 싶어" `CC`

**사용자 맥락**: 취미로 요리 영상을 올리던 크리에이터가 채널을 본격적으로 키우고 싶다. outcome(구독자 수? 수익? 브랜드 인지도?), metric(어떤 수치가 성공인가?), boundary(투자 가능 시간/비용?), priority(성장 속도 vs 콘텐츠 품질?) 중 2개 이상이 미정이다.

**트리거 발화**: "채널을 좀 제대로 키워보고 싶다" (정의 자체 불확실)

**호출 예시**:
```
/goal 요리 유튜브 채널을 키우고 싶은데, 구독자 수/수익/브랜드 중 뭘 목표로 잡아야 할지 모르겠어.
```

**결핍(전)**: "something like 성장"이라는 막연한 의도를 DefinedEndState로 구체화해야 한다.

**해결(후)**: AI가 후보 목표들을 제안하고 사용자가 Accept/Modify/Reject로 "6개월 내 구독자 1만, 주 2회 업로드, 품질 우선" GoalContract를 확정한다.

**인접 프로토콜 배제**: **Hermeneia** — 표현이 모호한 게 아니라 목표 자체가 미정이다.

---

### 5. "뭔가 교육 콘텐츠를 만들어보고 싶은데" `CC`

**사용자 맥락**: 현직 마케터가 자신의 전문성으로 교육적인 걸 해보고 싶다. 온라인 강의? 유튜브 튜토리얼? 뉴스레터? 워크숍? 형식조차 정해지지 않았고, 타겟 수강생, 무료/유료 여부, 성공 기준 모두 미정이다.

**트리거 발화**: "뭔가 교육 콘텐츠를 만들어보고 싶은데" (형식조차 미정, "something like" 전형)

**호출 예시**:
```
/goal 마케팅 경험을 살려서 교육 콘텐츠를 만들고 싶은데, 강의/튜토리얼/뉴스레터 중 뭘 할지부터 정해야 할 것 같아.
```

**결핍(전)**: Outcome/Metric/Boundary/Priority 모든 차원이 열려 있다. "not sure what I want"의 전형이다.

**해결(후)**: AI가 후보 목표들을 제안하고 사용자가 형태를 잡아가는 Telos의 전형적 시나리오. "주니어 마케터 대상 유료 온라인 강의" 같은 GoalContract로 구체화한다.

**인접 프로토콜 배제**: **Epitrope** — 권한 위임 문제가 아니라 무엇을 달성할지의 정의 부재가 핵심이다.
