# Syneidesis

결정 지점에서 잠재적 갭 표면화 (συνείδησις: 함께 앎)

> [English](./README.md)

## Syneidesis란?

그리스어 συνείδησις(함께 앎)의 현대적 재해석으로, **AI를 "보조적 인식 주체(auxiliary epistemic agent)"**로 위치시키는 프로토콜이다.

### 철학적 의미

전통적 양심(내적 증인)을 외재화된 대화 파트너로 변환. 인식적 분업—AI는 패턴 인식, 인간은 가치 판단—을 구조화.

### AI 정렬 관점

"약한 AI" 시대의 적절한 인간-AI 협업 모델. 수정가능성(corrigibility)을 구체적 행동 규칙으로 번역하고, 보수적 실패 모드(비가역적 행동 대기)를 구현.

### 상호작용 설계 관점

선택적 마찰(intentional friction)의 정교한 구현. `GapUnnoticed → AuditedDecision` 변환을 UX 흐름으로 물질화.

## 프로토콜 흐름

```
Phase 0: Detection    → 결정 지점의 갭 탐지 (비표시)
Phase 1: Surfacing    → 갭 선택 및 제시 (call AskUserQuestion)
Phase 2: Adjustment   → 응답을 상태에 반영
```

## 핵심 원칙

**Surfacing over Deciding**: AI가 "더 나은 결정을 내리는 것"이 아니라 **"사용자가 더 나은 결정을 내릴 수 있도록 인지적 사각지대를 조명하는 것"**을 목표로 한다.

## 사용자 역할

**인지가 아닌 판단**: 갭이 표면화되면 그 관련성을 평가합니다 — 해결하거나, 무시하거나, 보류합니다. AI는 표면화하고, 사용자가 판단합니다.
