# Syneidesis 사용 시나리오

> 실사용 전제 — 이 시나리오들은 사용자가 해당 프로토콜을 의식적으로 호출(`/gap`)하는 상황을 기술합니다.
> Skill이 세션에 로드되지 않으면 충돌이 발생하지 않습니다.

**프로토콜**: Syneidesis (`/gap`)
**결핍 → 해소**: `GapUnnoticed → AuditedDecision`

---

### 1. DB 마이그레이션 배포 직전 `SW`

**사용자 맥락**: 사용자 테이블에 NOT NULL 컬럼을 추가하는 마이그레이션을 프로덕션에 적용하려 한다. AI가 마이그레이션 SQL을 작성 완료하고 실행하려는 순간.

**트리거 발화**: "마이그레이션 준비 끝났으니 프로덕션에 바로 적용하자" (비가역적 데이터 변경, committed action)

**호출 예시**:
```
/gap NOT NULL 컬럼 추가 마이그레이션을 프로덕션에 적용하려 한다. 기존 행의 기본값 처리, 롤백 전략, 테이블 락 시간을 점검해 줘.
```

**결핍(전)**: 커밋된 상태 변경 액션(DB 스키마 변경)에 대해 기존 행의 기본값 처리, 롤백 전략, 테이블 잠금 시간 등 관찰 가능한 gap이 존재한다.

**해결(후)**: AI가 "기존 행의 기본값 처리 전략은 확인됐나요?", "테이블 락 시간 추정은?" 등 질문으로 gap을 표면화하고, 사용자 응답에 따라 AuditedDecision으로 전환한다.

**인접 프로토콜 배제**: **Aitesis** — Aitesis는 실행 전 맥락 부족을 탐지하지만, 여기서는 구체적인 코드 변경이 이미 완료되었고 실행(커밋) 직전의 gap 감사가 필요하다.

---

### 2. K8s 리소스 제한 변경 `kubectl apply` `SW`

**사용자 맥락**: 프로덕션 Pod의 CPU/메모리 limits를 조정하는 매니페스트 변경을 적용하려 한다. AI가 YAML 수정을 완료하고 `kubectl apply`를 실행하려는 순간.

**트리거 발화**: "YAML 수정 끝났으니 kubectl apply 실행하자" (인프라 상태 직접 변경)

**호출 예시**:
```
/gap 프로덕션 Pod CPU/메모리 limits 조정 YAML을 kubectl apply 하려 한다. HPA 동작 변경, 리소스 경쟁, OOMKill 임계값을 점검해 줘.
```

**결핍(전)**: HPA 동작 변경, 다른 Pod의 리소스 경쟁, OOMKill 임계값 등의 gap이 있다. 인프라 상태를 직접 변경하는 액션이라는 점에서 코드 레벨 변경과 구별된다.

**해결(후)**: AI가 HPA/리소스 연쇄 영향 등을 질문으로 표면화하고, 응답에 따라 진행/보류를 결정해 AuditedDecision으로 전환한다.

**인접 프로토콜 배제**: **Epitrope** — 누가 실행할지의 위임 문제가 아니라 어떤 검토 항목이 누락되었는지의 결정 갭 문제다.

### 3. 투자위원회 최종 의결 상정 `KW`

**사용자 맥락**: PE 심사역이 포트폴리오 회사 추가 투자(follow-on) 의결안을 투자위원회에 상정하려 한다. 상정안은 완성되어 있고, 의결이 통과되면 자금이 집행된다.

**트리거 발화**: "상정안 완성됐으니 투자위원회에 올리자" (자금 집행 결정)

**호출 예시**:
```
/gap follow-on 투자 의결안을 투자위원회에 상정하려 한다. 환율 헤지, 기존 투자자 희석 조건, 경쟁사 동일 라운드 참여 여부를 점검해 줘.
```

**결핍(전)**: 환율 헤지 전략 미기재, 기존 투자자 희석 조건 미검토, 경쟁사 동일 라운드 참여 여부 미확인 등 상정 직전에 누락된 gap이 있다.

**해결(후)**: AI가 절차적/가정/대안 gap을 질문으로 표면화하고, 응답에 따라 감사 가능한 의결 결정으로 만든다.

**인접 프로토콜 배제**: **Telos** — 목표(추가 투자 의결)는 이미 있고, 부족한 것은 목표 정의가 아니라 결정 전 고려항목 확인이다.

### 4. 연구 논문 저널 제출 `KW`

**사용자 맥락**: 연구자가 논문 초고를 Nature 계열 저널에 제출하기로 결정하고 제출 직전이다.

**트리거 발화**: "논문 준비 끝났으니 제출하자" (학술 외부 공개)

**호출 예시**:
```
/gap Nature 계열 저널에 논문을 제출하려 한다. 공동저자 기여도 확인서, 표본 크기 가정, 선행연구 차별점, 저널 선택 대안을 점검해 줘.
```

**결핍(전)**: 공동저자 기여도 확인서 미첨부(절차), 통계 검정력 분석의 표본 크기 가정 미기재(가정), 선행연구 2편에 대한 차별점 미기술(고려사항), 오픈액세스 vs. 구독형 대안 저널 미검토(대안) 등 gap이 표면화되지 않은 상태다.

**해결(후)**: AI가 절차적/방법론적 gap을 질문으로 표면화하고, 답변 반영 여부를 추적해 제출 결정을 감사 가능한 상태로 만든다.

**인접 프로토콜 배제**: **Prothesis** — 관점 선택 문제가 아니라 이미 정한 실행 직전의 숨은 가정 점검 문제다.

---

### 5. 크리에이터 협업 계약서 서명 `CC`

**사용자 맥락**: 두 유튜버가 공동 채널을 런칭하기로 하고 계약서에 서명하려 한다. 서명은 법적 구속력이 있는 행위이다.

**트리거 발화**: "계약서 다 봤으니 서명하자" (법적 구속력 있는 행위)

**호출 예시**:
```
/gap 공동 채널 계약서에 서명하려 한다. 수익 분배 비율, IP 소유권, 채널 탈퇴 시 구독자/콘텐츠 처리, 브랜드 딜 수락 기준을 점검해 줘.
```

**결핍(전)**: 수익 분배 비율, IP 소유권(공동 제작 콘텐츠의 개별 사용 권한), 채널 탈퇴 시 구독자/콘텐츠 처리, 브랜드 딜 수락 기준, 업로드 주기 미이행 시 조항 등에서 인지하지 못한 갭이 존재할 수 있다.

**해결(후)**: AI가 IP/수익분배/탈퇴 조건 등 gap을 질문으로 표면화하고, 사용자 판단을 기록해 감사 가능한 계약 결정으로 만든다.

**인접 프로토콜 배제**: **Epitrope** — 누가 실행할지의 위임 문제가 아니라 어떤 검토 항목이 누락되었는지의 결정 갭 문제다.
