# Aitesis 사용 시나리오

> 실사용 전제 — 이 시나리오들은 사용자가 해당 프로토콜을 의식적으로 호출(`/inquire`)하거나, AI가 맥락 부족을 감지하여 활성화하는 상황을 기술합니다.
> Skill이 세션에 로드되지 않으면 충돌이 발생하지 않습니다.

**프로토콜**: Aitesis (`/inquire`)
**결핍 → 해소**: `ContextInsufficient → InformedExecution`

---

### 1. 처음 보는 코드베이스 버그 수정 `SW`

**사용자 맥락**: 새로 합류한 프로젝트에서 결제 모듈의 환불 버그 수정을 요청받았다. AI가 실행 계획을 세우려 하지만, 결제 게이트웨이 연동 방식, 환불 상태 머신, 관련 테스트 컨벤션 등 코드베이스 고유의 맥락이 부재하다.

**트리거 발화**: "결제 모듈의 환불 버그를 수정해 달라" (실행 계획 존재 + 코드베이스 맥락 부재)

**호출 예시**:
```
/inquire 결제 모듈의 환불 버그를 수정해야 하는데, 이 코드베이스의 결제 게이트웨이 연동 방식과 환불 상태 머신을 먼저 파악해 줘.
```

**결핍(전)**: 코드베이스 고유의 맥락이 부재. AI가 먼저 codebase를 조사(self-investigation)한 뒤 해소되지 않는 gap만 사용자에게 질문한다.

**해결(후)**: AI가 코드/문서를 먼저 조사하고, 남은 블로킹 갭(예: 환불 상태 머신의 비즈니스 규칙)만 AskUserQuestion으로 수집해 실행 계획을 업데이트한다.

**인접 프로토콜 배제**: **Prothesis** — 분석 관점이 아니라 실행에 필요한 구체적 맥락 정보가 부족한 상황이다.

### 2. 외부 API(Stripe) 연동 코드 작성 `SW`

**사용자 맥락**: Stripe webhook을 처리하는 엔드포인트 구현 요청. AI가 일반적인 Stripe webhook 구현은 알지만, 이 프로젝트의 Stripe 계정 설정(API 버전, webhook signing secret 위치, 이벤트 타입 필터), 기존 에러 핸들링 패턴, 멱등성 처리 방식 등의 맥락이 부재하다.

**트리거 발화**: "Stripe webhook 엔드포인트를 만들어 달라" (외부 의존성 + 프로젝트별 설정 gap)

**호출 예시**:
```
/inquire Stripe webhook 엔드포인트를 만들어야 하는데, 프로젝트의 Stripe 설정(API 버전, signing secret 위치, 이벤트 필터)을 먼저 확인해 줘.
```

**결핍(전)**: 외부 의존성(Stripe)이 핵심. 프로젝트 고유 설정 없이 구현하면 재작업 필수.

**해결(후)**: AI가 CI 설정, 환경변수, 기존 코드를 먼저 조사하고, 남은 갭(signing secret 위치, 이벤트 필터)만 질문해 InformedExecution으로 전환한다.

**인접 프로토콜 배제**: **Syneidesis** — 특정 실행의 커밋된 액션에 대한 gap 감사가 아니라, 실행 자체를 시작하기 전에 맥락이 부족한 상황이다.

### 3. 규제 변경 대응 절차 업데이트 `KW`

**사용자 맥락**: 금융 컴플라이언스 담당자가 새로운 개인정보보호법 시행령에 맞게 고객 데이터 처리 절차를 업데이트해 달라고 요청한다. 현재 절차 문서 위치, 적용 대상 조항 범위, 기존 법무팀 검토 의견 유무, 시행 일자와 유예 기간 등 다수의 context가 부재하다.

**트리거 발화**: "새로운 개인정보보호법 시행령에 맞게 절차를 업데이트해 줘" (규제 도메인 + 내부 절차 이중 gap)

**호출 예시**:
```
/inquire 개인정보보호법 시행령 대응으로 데이터 처리 절차를 업데이트해야 하는데, 현재 절차 문서 위치와 법무팀 기검토 여부부터 확인이 필요해.
```

**결핍(전)**: 규제 도메인 + 내부 절차라는 이중 gap. 자체 해결 불가능한 내부 정보가 핵심.

**해결(후)**: AI가 먼저 자체 조사(시행령 공개 자료 탐색)를 시도한 후, 자체 해결 불가능한 gap(내부 절차 문서 위치, 법무팀 기검토 여부)만 사용자에게 묻는다.

**인접 프로토콜 배제**: **Telos** — 목표(규제 대응)는 명확하며 부족한 것은 실행 사실 컨텍스트다.

### 4. 의료 AI 뉴스레터 첫 에디션 `CC`

**사용자 맥락**: 의료 AI 관련 뉴스레터를 시작하겠다고 결정한 크리에이터가 첫 에디션 작성을 요청했다. 규제 환경(FDA 승인 프로세스), 타겟 독자 전문성 수준(의사 vs 헬스케어 경영진 vs 일반인), 경쟁 뉴스레터 현황, 인용 시 필수 면책 조항 등 실행에 필수적이지만 컨텍스트에 부재한 도메인 지식이 있다.

**트리거 발화**: "의료 AI 뉴스레터 첫 에디션을 작성해 줘" (novel domain, 규제/면책 맥락 필수)

**호출 예시**:
```
/inquire 의료 AI 뉴스레터 첫 에디션을 작성해야 하는데, 타겟 독자 수준(의사/경영진/일반인)과 면책 조항 요구사항을 먼저 파악해 줘.
```

**결핍(전)**: novel domain 트리거. 의료 AI는 규제, 면책, 독자 전문성 수준 등 도메인 고유 맥락이 필수.

**해결(후)**: AI가 공개 자료를 조사한 뒤, 남은 블로킹 갭(타겟 독자 수준, 면책 조항 요구사항)을 구조화해 사용자에게 질의한 뒤 실행한다.

**인접 프로토콜 배제**: **Hermeneia** — 발화 표현은 명확하고, 문제는 의도가 아니라 실행 데이터 결손이다.

### 5. 다국어 강의 현지화 `CC`

**사용자 맥락**: 프로그래밍 강사가 강의를 일본어로 현지화해 달라고 요청했다. 단순 번역이 아니라 일본 개발자 커뮤니티의 기술 용어 관행(카타카나 vs 영어 원문), 일본 교육 콘텐츠 시장의 가격대, 현지 경쟁 강의 현황, 결제 시스템(일본 내 주요 플랫폼) 등 현지화 실행에 필수적인 맥락이 다수 부재한다.

**트리거 발화**: "내 강의를 일본어로 현지화해 줘" (번역 vs 현지화 scope + 현지 시장 맥락)

**호출 예시**:
```
/inquire 프로그래밍 강의를 일본어로 현지화해야 하는데, 일본 개발자 커뮤니티의 기술 용어 관행과 현지 플랫폼/가격대를 먼저 조사해 줘.
```

**결핍(전)**: ambiguous scope(번역 vs 현지화) 트리거에도 해당. 현지 시장 맥락 없이는 의미 있는 현지화가 불가능하다.

**해결(후)**: AI가 일본 교육 플랫폼 정보를 조사한 뒤, 남은 갭(가격 전략, 용어 관행, 결제 시스템)을 one-at-a-time으로 질문해 실행 맥락을 확정한다.

**인접 프로토콜 배제**: **Epitrope** — 허용 권한이 문제가 아니라, 실행에 필요한 객관적 사실이 빠져 있다.
