### 인사이트 ①: 리뷰 기반 반복 정제 패턴

이 프로젝트는 단순한 "기능 구현 → 다음 기능" 직선형 개발이 아니다. 전형적인 사이클은:

```
feat(구현) → PR 생성 → fix(리뷰 반영) → fix(2차 리뷰) → fix(추가 findings) → merge
```

Prothesis/Epitrope 분리 작업(#68)이 대표적:
- `feat(epitrope): expand formal block` → `fix: 코드 리뷰 findings (v4.0.1)` → `fix: 추가 리뷰 findings (v4.0.2)` → `fix: 2차 리뷰 findings (v4.0.3)`

**해석**: 사용자는 **"한 번에 완벽하게" 보다 "빠르게 올리고 반복 정제"** 전략을 채택. Claude Code를 리뷰어로 활용하여 피드백 루프를 자동화한 것으로 보임.

## 11. 종합 인사이트: 사용자 프로파일

### "형식주의적 반복 정제자" (Formalist Iterative Refiner)

| 차원 | 성향 |
|------|------|
| **작업 리듬** | 오전 집중 + 심야 몰입의 이원 구조, 토요일 폭발 |
| **품질 전략** | 빠른 초안 → PR 리뷰 → 반복 수정 (fix:feat = 1.85:1) |
| **설계 철학** | 형식 명세 우선 (formal block, typed deficit→resolution), 구조적 정합성 집착 |
| **도구 활용** | Claude Code를 리뷰어·검증자로 적극 활용, /verify 게이트 |
| **명명 감각** | 그리스어 고유명사 + 영어 타입 시스템 + 한국어 설명의 3중 레이어 |
| **확장 전략** | 중심(Prothesis)에서 실험 → 검증된 패턴을 주변으로 확산 |
| **완결 의지** | 21일간 148커밋, 7개 프로토콜 + 2개 스킬, 후반으로 갈수록 가속 |

### 핵심 메타 패턴

1. **정합성 집착(Consistency Obsession)**: 커밋 컨벤션, co-change 패턴, 정적 검사 — 모든 계층에서 일관성을 강제하는 시스템을 구축
2. **자기 참조적 개선(Self-Referential Improvement)**: 프로토콜이 인식론적 결핍을 해소하듯, 개발 프로세스 자체도 결핍(version-staleness, tool-grounding 누락)을 탐지하고 해소하는 구조
3. **AI 협업의 내재화**: PR 리뷰의 56%가 AI 피드백 반영 — Claude Code가 "페어 프로그래머"가 아닌 **"상시 감사관(auditor)"** 역할로 통합됨
