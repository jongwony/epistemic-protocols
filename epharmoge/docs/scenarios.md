# Epharmoge 사용 시나리오

> 실사용 전제 — 이 시나리오들은 AI가 실행 결과의 적용 맥락 불일치를 감지하여 활성화하거나,
> 사용자가 결과 적합성을 의식적으로 호출(`/contextualize`)하는 상황을 기술합니다.
> Skill이 세션에 로드되지 않으면 충돌이 발생하지 않습니다.

**프로토콜**: Epharmoge (`/contextualize`)
**결핍 → 해소**: `ApplicationDecontextualized → ContextualizedExecution`

---

### 1. 기술적으로 정확한 아키텍처 제안의 맥락 부적합 `SW`

**사용자 맥락**: AI가 프로토콜 시스템에 worktree 격리와 병렬 worker 배포를 추가하는 제안을 했다. 제안 자체는 기술적으로 정확하고 빌트인 도구(`/batch`)에서 검증된 패턴이었으나, 프로젝트의 설계 원칙(인식론적 도구와 실행 도구의 직교적 분리)과 충돌한다.

**트리거 조건**: `correct(R) ∧ ¬warranted(R, X)` — 제안이 정확하지만 맥락에 맞지 않음

**호출 예시**:
```
/contextualize 방금 제안한 아키텍처 변경이 프로젝트의 실제 사용 패턴과 설계 원칙에 맞는지 확인해 봐.
```

**결핍(전)**: AI가 기술적 정확성만으로 제안의 적합성을 판단. 빌트인 실행 도구의 패턴(worktree 격리)을 인식론적 프로토콜에 그대로 적용하는 "mirroring" 오류.

**해결(후)**: 3가지 불일치 식별 — (1) 미러링 vs 공존 원칙 위반, (2) 선형 파이프라인 가정 vs 실제 모듈식 조합 패턴, (3) 빌트인 도구 scope 경계 침범. 제안을 철회하고 "Coexistence over Mirroring" 원칙을 도출.

**인접 프로토콜 배제**: **Syneidesis** — 결정 전 gap 감사가 아니라, 이미 완료된 분석 결과의 적용 적합성 검증이다.

### 2. 로컬 환경 기반 설정의 프로덕션 부적합 `SW`

**사용자 맥락**: AI가 개발자의 로컬 환경에서 잘 동작하는 Docker Compose 설정을 작성했다. 볼륨 마운트 경로, 환경 변수 참조, 리소스 제한이 모두 개발 환경 기준이다. 프로덕션 배포(Kubernetes)에서는 동일한 설정이 작동하지 않는다.

**트리거 조건**: 결과(R)가 기술적으로 정확하지만 적용 맥락(X = production)에서 보증되지 않음

**호출 예시**:
```
/contextualize 이 Docker Compose 설정이 프로덕션 K8s 환경에서도 적합한지 확인해 줘.
```

**결핍(전)**: 개발 환경에서 검증된 설정을 프로덕션에 그대로 적용하려는 맥락 전이 오류. `./data:/app/data` 같은 호스트 볼륨 마운트, `restart: always` 같은 단일 노드 가정.

**해결(후)**: 환경 가정 불일치(호스트 볼륨 → PersistentVolumeClaim), 규모 가정 불일치(단일 노드 → 멀티 노드), 시크릿 관리 불일치(환경 변수 직접 참조 → K8s Secret) 식별. 적응된 설정 제안.

**인접 프로토콜 배제**: **Aitesis** — 실행 전 맥락 부족이 아니라, 이미 완성된 결과물의 적용 환경 불일치다.

### 3. 해외 마케팅 전략의 현지 관행 부적합 `KW`

**사용자 맥락**: AI가 미국 시장 데이터를 기반으로 SaaS 제품의 마케팅 전략을 수립했다. 가격 모델(연간 결제 할인), 결제 수단(신용카드 중심), 콘텐츠 채널(LinkedIn + 블로그)이 모두 미국 SaaS 관행을 따른다. 실제 타겟 시장은 일본이다.

**트리거 조건**: convention mismatch — 전략이 미국 관행에서 정확하지만 일본 시장 관행과 불일치

**호출 예시**:
```
/contextualize 이 마케팅 전략이 일본 SaaS 시장의 관행과 부합하는지 확인해 줘.
```

**결핍(전)**: 미국 중심 SaaS 관행을 일본에 그대로 적용. 일본 B2B의 특성(월간 결제 선호, 청구서 결제, 전시회/세미나 중심 리드 생성)을 고려하지 않음.

**해결(후)**: 가격 모델 불일치(연간 할인 → 월간 결제 기본), 결제 수단 불일치(신용카드 → 청구서/은행 이체), 채널 불일치(LinkedIn → 전시회/파트너 채널) 식별. 일본 시장에 맞는 적응 전략 제안.

**인접 프로토콜 배제**: **Prothesis** — 분석 관점이 부재한 것이 아니라, 정확한 분석이 잘못된 맥락에 적용된 상황이다.

### 4. 학술 논문 스타일의 비즈니스 보고서 부적합 `KW`

**사용자 맥락**: AI가 시장 분석 보고서를 작성했다. 분석의 깊이와 정확성은 우수하나, 형식이 학술 논문 스타일(문헌 리뷰, 방법론 섹션, 인용 표기)로 작성되었다. 실제 사용 맥락은 C-level 경영진 대상 의사결정 보고서다.

**트리거 조건**: scope overflow — 분석 범위가 적용 맥락의 요구를 초과

**호출 예시**:
```
/contextualize 이 보고서가 C-level 경영진이 의사결정에 바로 사용할 수 있는 형식인지 확인해 줘.
```

**결핍(전)**: 학술적 엄밀성이 비즈니스 의사결정 맥락에서 오히려 방해. 15페이지 분석을 C-level이 읽을 시간이 없다.

**해결(후)**: 형식 불일치(학술 → executive summary + key findings), 깊이 불일치(방법론 상세 → 결론 중심), 액션 부재(분석만 → 권고 사항 + 다음 단계) 식별. 동일 분석 내용을 경영진 보고서 형식으로 적응.

**인접 프로토콜 배제**: **Hermeneia** — 사용자 의도는 명확(시장 분석 보고서). 문제는 의도가 아니라 결과의 적용 형식이다.

### 5. 범용 교육 콘텐츠의 플랫폼별 부적합 `CC`

**사용자 맥락**: AI가 프로그래밍 입문 강의 스크립트를 작성했다. 내용은 정확하고 교육적으로 우수하나, 45분 분량의 강의실 강의 형식으로 작성되었다. 실제 게시 플랫폼은 YouTube이며, 시청자 평균 시청 시간은 8분이다.

**트리거 조건**: environment assumption — 강의실 환경을 가정했지만 실제 맥락은 YouTube

**호출 예시**:
```
/contextualize 이 강의 스크립트가 YouTube 시청 패턴에 맞는 형식인지 확인해 줘.
```

**결핍(전)**: 강의실 관행(45분 연속 강의, 순차적 개념 빌드업, Q&A 시간)이 YouTube 관행(8분 시청, hook-first 구조, 챕터 마커)과 충돌.

**해결(후)**: 시간 불일치(45분 → 8-12분 에피소드 분할), 구조 불일치(순차 빌드업 → hook + payoff), 상호작용 불일치(Q&A → 댓글 유도 + 퀴즈 삽입) 식별. 동일 내용을 YouTube 시리즈 형식으로 적응.

**인접 프로토콜 배제**: **Telos** — 목표(프로그래밍 입문 교육)는 명확하다. 문제는 완성된 결과의 게시 맥락 적합성이다.
