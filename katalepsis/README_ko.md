# Katalepsis

AI 작업에 대한 확실한 이해 달성 (κατάληψις: 확고하게 파악함)

> [English](./README.md)

## Katalepsis란?

스토아 철학의 κατάληψις (확고한 파악, 확실한 이해)를 현대적으로 재해석한 프로토콜 — **AI가 생성한 작업을 사용자가 따라가며 검증된 이해에 도달**할 수 있게 합니다.

### 핵심 문제

AI가 복잡한 작업을 완료했을 때, 사용자는 종종 자신이 무엇을 이해하지 못하는지 모릅니다 (unknown knowns). 이해했다고 생각하거나, 어디서부터 시작해야 할지 몰라 변경사항에 압도당할 수 있습니다.

### 해결책

**설명보다 이해 확인**: 일방적으로 설명하는 대신, AI가 카테고리화된 진입점을 통해 구조화된 검증을 진행합니다. 사용자가 먼저 이해하고 싶은 부분을 선택하면, AI가 점진적인 질문을 통해 이해를 확인합니다.

### 단순 설명과의 차이

| 차원 | 단순 설명 | Katalepsis |
|------|----------|------------|
| 방향 | AI가 말하고, 사용자가 듣는다 | AI가 검증하고, 사용자가 확인한다 |
| 진입점 | AI가 무엇을 설명할지 결정 | 사용자가 집중할 영역 선택 |
| 확인 | 설명 후 가정됨 | 질문을 통해 검증됨 |
| 진행 상황 | 정의되지 않음 | 태스크로 추적됨 |

## 프로토콜 흐름

```
Phase 0: 카테고리화  → AI 작업 분석, 변경 카테고리 추출
Phase 1: 진입점     → 카테고리 제시, 사용자 선택 (AskUserQuestion)
Phase 2: 등록       → 선택된 카테고리에 대한 태스크 생성 (TaskCreate)
Phase 3: 검증 루프  → 점진적으로 이해 확인 (AskUserQuestion + TaskUpdate)
```

## 사용 시점

**사용**:
- AI가 상당한 코드 변경을 완료한 후
- 사용자가 "뭘 했어?", "설명해줘", "이해하고 싶어"라고 할 때
- 복잡한 리팩토링, 새 기능, 아키텍처 변경

**건너뛰기**:
- 사소한 변경 (오타, 포매팅)
- 사용자가 이미 이해를 보여줌
- 사용자가 명시적으로 설명을 거부

## 갭 유형

| 유형 | 설명 |
|------|------|
| **Expectation** | 사용자가 가정한 동작이 실제와 다름 |
| **Causality** | 사용자가 왜 그렇게 되는지 이해하지 못함 |
| **Scope** | 사용자가 변경의 전체 영향을 보지 못함 |
| **Sequence** | 사용자가 실행 순서를 이해하지 못함 |
