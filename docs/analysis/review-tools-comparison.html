02축별 비교

축 /code-review review-pr /frame
결손 / 목적 git diff의 코드 정합성 버그 PR/diff의 다차원 품질 (테스트·타입·주석·에러) 탐구의 분석틀 부재 → FramedInquiry
차원 분해 6개 고정 차원, 항상 전부 병렬 6개 전문가, diff에 따라 조건부 선택 (기본 순차) 사용자가 선택한 N개 관점 렌즈, 격리 병렬
검증 모델 발견당 적대적 단일 검증자 (독립 agent, 기본 unclear, 5경로 반증) 없음 — 에이전트 자가 신뢰도 채점 (≥80) 검증이 아닌 교차 종합 + 변환 트레이스 입증
거짓 양성 통제 구조적: 2차 에이전트 + rejected 버킷 + slot 예산(25, high 항상 통과) 자가보고: 생산자가 곧 판정자 ("minimize FP" 만다트) 해당 없음 (정답형 검증이 아님)
상호작용 모델 자율 · 게이트 없음 자율 · 게이트 없음 3회 게이트 (turn yield) · 사용자 판단이 결과를 구성
산출물 랭킹·dedup 리포트 + --comment 인라인 PR 코멘트 수동 집계 markdown 요약 (인라인 없음) FramedInquiry 객체 (렌즈 핸드오프 / 합성 Lens)
수렴 / 종료 검증·예산 버킷 처리 후 Report에서 종료 전 에이전트 보고 집계 후 종료 사용자 만족 / withdraw / Esc + 변환 트레이스 입증
코드 수정 보고만 (mutating 없음) code-simplifier가 재작성 (통과 후) 해당 없음
확장성 바이너리 컴파일 — 편집 불가 markdown 명령 + 에이전트 파일 — 완전 포크 가능 SKILL.md 포크 가능
Initiator 사용자 호출 → AI 완주 사용자 호출 → AI 완주 AI 배치 / 사용자 선택 (Detection with Authority)

03장단점

/code-review
장점
  • 생산자/판정자 분리한 독립 적대 검증 — 거짓 양성에 구조적으로 강함
  • 정형 dedup + severity/confidence 랭킹
  • high/critical은 예산 무시하고 항상 검증
  • 원격 PR 모드 + --comment 인라인 게시
단점
  • 6차원 고정 — 끌 수도 늘릴 수도 없음
  • 바이너리 컴파일 — 프롬프트 편집 불가
  • 코드를 고치지 않고 보고만
  • effort→예산 매핑이 불투명 (휴리스틱+상수)
review-pr
장점
  • 전문가 폭: 에러 핸들링·타입 설계(1-10점)·주석·테스트 전용 에이전트
  • diff에 따라 조건부 선택 — 불필요한 패스 생략
  • 리뷰 후 code-simplifier가 코드 재작성
  • 전부 편집/포크 가능한 markdown
단점
  • 독립 검증자 없음 — 생산자가 곧 판정자(자가 채점)라 FP에 약함
  • dedup 키 없음 (수동 집계)
  • 이름과 달리 인라인 PR 코멘트 미게시
  • 기본 순차라 느릴 수 있음 (병렬 opt-in)
/prothesis:frame
장점
  • 코드가 아닌 사고의 틀을 다룸 — 적용 영역이 다름
  • 핵심 판단을 사용자가 구성 (인간 주체성 보존)
  • 변환 트레이스로 수렴을 입증(주장 아님)
  • 다른 프로토콜과 조합 가능
단점
  • 3회 게이트 — 자동화/무인 실행엔 부적합
  • 코드 결함 탐지가 목적이 아님 (리뷰 도구 대체 불가)
  • 사용자 참여 없이는 진행 불가
  • 러닝 커브 (형식 블록·게이트 의미론)

04언제 무엇을 — 판정

세 도구는 중복이 아니라 서로 다른 니치다. code-review와 review-pr은 "같은 아이디어의 두 세대"(엄격한 검증 엔진 vs 확장 가능한 전문가 셋), frame은 아예 다른 층위(코드 결함 vs 사고틀 부재)다.

/code-review거짓 양성이 비싼 머지 전 정합성 게이트. 적대 검증자가 값을 함. 인라인 PR 코멘트가 필요할 때.
review-pr차원 특화 심층 감사(에러 핸들링·타입 불변식·테스트 갭·comment rot), 리뷰어 셋 커스터마이즈, 리뷰 후 단순화 패스가 필요할 때.
/frame코드가 아니라 문제를 어떤 틀로 볼지가 막혔을 때. 분석 렌즈를 사용자가 골라 다관점으로 탐구하고 싶을 때.
부하지지 차이 한 줄 — code-review의 agent('Adversarially verify: …') · "do NOT default to confirmed" (독립 검증) vs review-pr의 code-reviewer.md · "Only report issues with confidence ≥ 80" (자가 임계). 독립 검증 vs 자가 임계가 두 리뷰 도구를 가르는 핵심이고, 게이트 유무가 리뷰 도구 전체와 frame을 가른다.
