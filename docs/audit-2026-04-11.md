# Epistemic Audit — 2026-04-11

> 프로젝트의 존재 가치 및 철학·수학·현상학적 soundness에 대한 전수 감사.

## Audit 구조

**Architecture**: 2-track heavy-parallel audit

- **Track α (Mission/Vision)**: 프로젝트의 stated mission이 내부 정합성과 외부 challenger test(Popper + pragmatist)를 통과하는가
- **Track β (Soundness)**: 프로젝트의 machinery가 내부 framework(Category Theory + Type Theory + A1-A7 + Gadamer)와 외부 anchor(Martin-Löf + Husserl + Popper)로 평가 시 sound한가

**Execution mode**: Dual-stream ensemble — 각 분석 태스크에서 Claude subagent와 Codex(gpt-5.4, reasoning=high, read-only)가 병렬로 독립 분석, 결과를 교차 검증. Single-model self-validation bias 완화 및 Track β의 "반성적 비판" 스탠스 구현.

**Scope**: Epistemic Core
- Track α sources: `CLAUDE.md` overview, `README.md`/`README_ko.md`, `marketplace.json`, 10 plugin.json descriptions (28 소스 entry)
- Track β sources: 10 SKILL.md formal blocks, `.claude/rules/{axioms, derived-principles, architectural-principles, meta-principle}.md` (25 principles inventory)

**Task pipeline**: #1 α-sources → #2 α-internal → #3 α-external → #4 β-SKILL-sources → #5 β-rules-sources → #6 β-internal → #7 β-external → #8 cross-track synthesis → #9 entropy triage → #10 final report (10/10 tasks completed)

**Clarification lineage**: 이 audit은 `/clarify`(Hermeneia) 세션으로 시작하여 5단계 gap 해소(Coherence → Background → Expression → Precision → Emergent)를 거쳤고, 이후 `/attend`(Prosoche)로 execution risk-assessed 모드에서 수행됨.

## Executive Summary

**감사의 단일 core finding**:

> **프로젝트는 inscribed rigor보다 약 2배 rigor를 claim한다. Formal notation(`∈`, `→`, `∘`, "coproduct", Π-adjacent parameterization)과 philosophical vocabulary(hermeneutic circle, eidetic, kataleptic, synthesis)가 대응하는 formal/philosophical grounding 없이 사용된다.**

이 관찰은 Claude stream과 Codex stream이 **독립적으로 수렴**한 메타-finding이며, audit에서 발견된 12개 strong-severity findings는 모두 이 한 가지 구조 이슈의 manifestation이다.

**그러나 동시에**:

> **Protocol layer는 axiom layer보다 강하다.** Local protocol state(Λ, Σ records), A1-A7 guard inscription discipline, anti-mutation rules, convergence evidence traces는 external anchors 하에서 실제로 robust하다. "Pre-formal vocabulary" 문제는 axiom layer와 meta-level claims에 국한된다.

**즉**, 프로젝트의 **실행 층은 working formal system**이나, **자기 설명 층은 그 formal status를 overclaim**한다. Audit의 권고는 이 gap을 인정(relabel), 보강(formalization), 또는 분리(hybrid) 방향으로 해결하는 것이다.

## Track β: Soundness 분석 요약

Track β는 internal framework와 external anchor 양쪽에서 매우 substantive한 findings를 생산했다.

### 내부 framework (Task #6) 주요 finding

- **`*` COMPOSITION block은 type-theoretic placeholder**: 10개 SKILL.md 모두에 `(D₁ × D₂) → (R₁ × R₂)` 표기가 있으나 D/R이 protocol TYPES에 binding되지 않음. Projection π₁/π₂, universal property 부재
- **A1-A7 inscription은 "remarkably self-consistent"**: Rule 2 (A1), Rule 13 (A6), Rule 14 (A3), Rule 15 (A7) 등 near-verbatim guard fixture가 10 SKILL.md에 일관 inscribed — **positive finding**
- **D7 Loop Continuity under Bounded Regret은 admitted non-derivation**: 파일 스스로 "principle's own contribution, not a direct reading of A2 or A5" 선언
- **Prothesis A7 guard consistency violation** (Codex unique): `prothesis/skills/frame/SKILL.md:248, 250, 555`에서 Phase prose "Do NOT skip"과 ELIDABLE CHECKPOINTS "elidable" 모순. A7이 명시적으로 경고한 "guard consistency" 패턴을 Prothesis 자신이 위반

### 외부 anchor (Task #7) 주요 finding

- **Martin-Löf TT**: TYPES block의 `∈ {...}` 표기는 set membership predicate이지 inductive sum type이 아님. 생성자 없음(`inl/inr`, labeled injection 부재). A7의 "TYPES coproduct classification" 주장은 존재하지 않는 생성자에 의존
- **Husserl eidetic reduction**: Deficit Empiricism N≥3은 Millian enumerative induction — phenomenological bracketing 후 생존 불가. "Empiricism" 용어의 phenomenological 용법 오용
- **Popper per-axiom**:
  - A1: 반증 조건 inscribed 안됨
  - A2: Standing/Active authority로 immunized (Popperian immunizing stratagem)
  - **A4: Already partially refuted** — Prothesis Await의 platform-dependency가 outcome equivalence claim을 부분 반증. TOOL GROUNDING entry는 auxiliary hypothesis patch
  - A7: **Self-referentially unfalsifiable** — 반증 시도가 guard list 추가로 흡수됨, pure conventionalist twist

## Ensemble 방법론 검증

### 왜 dual-stream이 가치를 냈는가

- **Claude unique**: Katalepsis 철학적 비판 (Stoic cognitive grasp criterion 인식), "Shadow mission" 재구성, Popper/pragmatist inversion pattern, LOOP blocks의 imperative 의미 인식, "formal notation at 2x rigor" meta-observation

**만약 single-model이었다면**:
- Claude만: empirical drift evidence 놓침 (Codex의 EC plugin.json 관찰), local protocol state의 구체적 positive assessment 놓침
- Codex만: Katalepsis 고전철학 critique 놓침, "shadow mission" 명시적 재구성 놓침, Popper/pragmatist inversion 패턴 인식 놓침

**Convergence로 확보된 신뢰도**:
- 12개 strong-severity findings가 **양 스트림 독립 합의** — random noise 제거
- Severity disagreement(예: C3 register bridge에서 Claude strong, Codex weak)는 그 자체로 user judgment 신호

### 방법론적 관찰

1. **Claude는 taxonomy/philosophical 분석에 강함**. Codex는 empirical evidence와 specific citations에 강함. **상호 보완적**.
2. **양 스트림 모두 의견이 수렴할 때**는 거의 항상 deterministic 또는 high-confidence design-decision으로 triage 가능.
3. **양 스트림이 다른 관점에서 같은 core issue에 수렴할 때**(예: Claude의 "hidden mission inversion" + Codex의 "project boundary ambiguity" 모두 "What is this project?"의 변주) — 이는 진정한 근본 이슈의 signal.
4. **Single-model 분석이라면 root finding (pre-formal vocabulary)을 이만큼 명시적으로 articulate할 수 없었을 것** — 두 스트림의 독립적 수렴이 이 finding의 confidence를 제공.

### Scope 한계 (limitations)

이 audit의 한계:
- **Epistemic Core scope 한정**: `src/` landing page, CI workflow, 구현 스크립트는 audit 대상 제외
- **Heavy-parallel 1회 실행**: Rückkehr (iterative refinement)는 별도 사이클에서 수행 필요. 본 report의 recommendation은 첫 cycle의 output이며 iteration 하에서 refine될 수 있음
- **External anchor 3개 선택**: Martin-Löf / Husserl / Popper는 Hermeneia clarification에서 user가 선택한 3 domain anchor. 다른 anchor(예: Coq-style constructive type theory, Heidegger existential analytic, Wittgenstein language-game)는 본 audit 범위 외
- **Claude + Codex ensemble의 bias**: 둘 다 LLM인 이상 공통 바이어스(학습 데이터, RLHF 방향)를 공유. 실제 다양성은 model family가 다른 만큼 제한적
- **Session telemetry absent**: README의 "one turn vs hours" quantified claim에 대한 empirical 데이터는 audit 하에서 측정 불가 (observability layer 자체가 design-decision matrix의 한 항목)
