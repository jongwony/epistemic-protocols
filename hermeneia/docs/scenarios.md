# Hermeneia 사용 시나리오

> 실사용 전제 — 이 시나리오들은 사용자가 의식적으로 호출(`/clarify`)하거나, AI가 표현-의도 불일치를 감지하여 사용자 확인 후 활성화되는 상황을 기술합니다.
> Skill이 세션에 로드되지 않으면 충돌이 발생하지 않습니다.

**프로토콜**: Hermeneia (`/clarify`)
**결핍 → 해소**: `IntentMisarticulated → ClarifiedIntent`

---

### 1. "이 코드 좀 깔끔하게 해 줘"의 다의성 `SW`

**사용자 맥락**: 개발자가 리팩토링을 요청하며 "clean up this code"라고 말했다. 포매팅/린팅, 함수 분해, 디자인 패턴 적용, 불필요 코드 제거 등 5가지 이상 해석이 가능하다.

**트리거 발화**: "이 코드 좀 깔끔하게 해 줘" (기술 용어의 다의성)

**호출 예시**:
```
/clarify "이 코드 좀 깔끔하게 해 줘"가 포매팅인지, 함수 분해인지, 패턴 적용인지 명확히 하고 싶어.
```

**결핍(전)**: 사용자가 이미 무엇을 원하는지 알고 있지만 표현이 모호한 전형적 사례. "깔끔하게"가 포매팅, 함수 분해, 패턴 적용, 코드 제거 등 어느 것인지 특정되지 않았다.

**해결(후)**: Expression/Precision gap을 사용자 선택으로 좁히고, 구체적인 리팩토링 범위(예: "함수 분해 + 불필요 코드 제거")로 ClarifiedIntent를 만든다.

**인접 프로토콜 배제**: **Prothesis** — 프레임워크 선택이 아니라 사용자의 의도 자체가 불명확하다.

---

### 2. 스코프-참조 불일치 AI-detected 강트리거 `SW`

**사용자 맥락**: 개발자가 "결제 지연 문제 고쳐줘"라고 했지만 직전 대화는 `checkout frontend`, 현재 로그는 `payment webhook timeout`을 가리킨다.

**트리거 발화**: AI가 scope-reference mismatch를 감지 (강한 AI-detected 트리거)

**호출 예시**:
```
AI 감지 → "결제 지연이 프런트엔드 체크아웃 지연과 웹훅 타임아웃 중 어느 쪽을 가리키는지 명확히 할까요?" 확인 후 활성화
```

**결핍(전)**: 프런트 지연과 웹훅 타임아웃은 원인 체인이 달라 잘못 해석하면 디버깅 경로가 완전히 빗나간다.

**해결(후)**: 사용자가 "웹훅 재시도 큐 지연"으로 스코프를 재지정해 ClarifiedIntent를 만들고, 그 표현을 후속 프로토콜 입력으로 사용한다.

**인접 프로토콜 배제**: **Telos** — 목표 불명확이 아니라 요청 표현의 참조 불일치 문제다.

---

### 3. 컨설턴트 "디지털 전환 수준 평가" `KW`

**사용자 맥락**: 전략 컨설턴트가 고객사의 디지털 전환 수준 평가를 요청한다. '디지털 전환 수준'이 기술 인프라 성숙도인지, 조직 문화 변화인지, 비즈니스 모델 혁신인지에 따라 완전히 다른 분석이 나온다.

**트리거 발화**: "우리 고객사의 디지털 전환 수준을 평가해 줘" (전문 용어 범위 불일치)

**호출 예시**:
```
/clarify "디지털 전환 수준 평가"가 기술 인프라 성숙도인지, 조직 문화 변화인지, 비즈니스 모델 혁신인지 범위를 좁히고 싶어.
```

**결핍(전)**: 요청자 본인은 "비즈니스 모델 혁신 관점"을 의도했지만 표현이 이를 담지 못한 Expression gap이다.

**해결(후)**: Expression gap을 해소해 "비즈니스 모델 혁신 성숙도 평가"로 범위를 특정한 ClarifiedIntent를 만든다.

**인접 프로토콜 배제**: **Prothesis** — 프레임워크 부재가 아니라 이미 하나의 의도가 있으나 표현이 부정확한 상황이다.

### 4. "영상을 cinematic하게 만들어 줘" `CC`

**사용자 맥락**: 여행 유튜버가 편집자에게 AI를 통해 지시를 전달했다. "cinematic"이 컬러 그레이딩 변경, 레터박스 추가, 슬로모션 증가, BGM 교체, 나레이션 톤 변경 등 다수의 해석을 허용한다.

**트리거 발화**: "영상을 좀 더 cinematic하게 만들어 줘" (주관적 미학 용어)

**호출 예시**:
```
/clarify "cinematic하게"가 컬러 그레이딩인지, 레터박스인지, 슬로모션인지, BGM 교체인지 구체화하고 싶어.
```

**결핍(전)**: 크리에이터 자신도 "cinematic"의 정확한 의미를 명확히 표현하지 못한 상태다. 해석에 따라 편집 작업 방향이 완전히 변경된다.

**해결(후)**: Gap type 선택으로 "cinematic = 컬러 그레이딩 + 슬로모션"처럼 의도를 구체화한 ClarifiedIntent를 만든다.

**인접 프로토콜 배제**: **Telos** — 목표(더 좋은 영상)는 존재. 표현과 의도 사이의 갭이 핵심이다.

### 5. "이 팟캐스트를 블로그로 변환해 줘" `CC`

**사용자 맥락**: 팟캐스트 호스트가 에피소드 텍스트 변환을 요청했다. "변환"이 트랜스크립트 정리(구어체 유지), 블로그 포스트로 재작성(문어체 전환), 핵심 요약만 추출, SEO 최적화된 아티클 작성 중 어떤 것인지에 따라 결과물이 완전히 다르다.

**트리거 발화**: "이 팟캐스트 에피소드를 블로그로 변환해 줘" (동작 정의 불명확)

**호출 예시**:
```
/clarify "블로그로 변환"이 트랜스크립트 정리인지, 문어체 재작성인지, 핵심 요약인지, SEO 아티클인지 확인하고 싶어.
```

**결핍(전)**: "변환해 줘"라는 단순한 표현 안에 복수의 유효한 해석이 숨어 있다.

**해결(후)**: "변환 = SEO 최적화된 블로그 포스트로 재작성"처럼 의도를 특정한 ClarifiedIntent를 만든다.

**인접 프로토콜 배제**: **Prothesis** — 분석 관점이 부족한 게 아니라, 사용자 본인의 표현 자체가 미정렬 상태다.
