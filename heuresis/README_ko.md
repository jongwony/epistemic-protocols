# Heuresis — /ideate (εὕρεσις)

프레임-병렬 발산 생성으로 저확장 후보장을 넓힘 (εὕρεσις: 발견, 찾아냄, 고안)

> [English](./README.md)

## Heuresis란?

**아직 선택이 일어나기 전, 후보장 자체가 얇은 순간**을 위한 프로토콜입니다 — 비어 있거나, 근본 목표가 아직 열려 있는데도 하나로 조기 수렴한 상태. 호출 발화만으로 백지인지 이미 조각을 갖고 있는지 추론하고(진입 질문 없음), 백지라면 구체적 아이디어를 보여주기 전에 추상 프레임을 먼저 펼쳐 사용자가 고르게 하며, 사용자가 연 프레임들 위에서 병렬로 후보를 생성합니다 — 그중 무엇도 탈락시키거나 순위 매기거나 고르지 않습니다. 멈추는 시점은 전적으로 사용자 몫입니다.

### 핵심 문제

어떤 후보장은 저확장 상태입니다(`CandidateFieldUnderexpanded`): 목표는 충분히 분명한데 객체-수준 대안이 비어 있거나 조기 수렴했습니다. 발산적 사고와 수렴적 사고는 서로 다른 인지 작업이며, 이 둘을 한 호흡에 섞는 것 — 생성과 판단을 동시에 하는 것 — 이 후보장을 얇게 유지하는 원인입니다. 여기에 두 위험이 겹칩니다: 구체 예시를 너무 일찍 보여주면 정작 촉발하려던 그 발상 자체를 고착시키고, AI 보조 생성은 별도의 장치 없이는 산출물들이 서로 수렴하게 만들 수 있습니다.

### 해법

**선택이 아닌 발산(Divergence over Selection)**: 발화만으로 백지/씨앗 진입을 추론한 뒤 갈립니다 — 백지 진입은 구체 아이디어가 존재하기 전에 추상 프레임 지도를 먼저 봅니다(소유감이 유지되고, 초기 발상을 고착시킬 것이 없습니다); 씨앗 진입은 이미 준 것에서 바로 확장합니다(직접 공급한 재료에는 같은 고착 위험이 적용되지 않으므로). 매 라운드는 열린 프레임들 위에서 병렬로 후보를 생성하며, 모든 후보에 출처(사용자 또는 AI)를 태깅하고, 아무것도 폐기하거나 순위 매기지 않습니다. 고정 라운드 할당량은 없습니다 — 후보장의 경계를 긋는 것은 사용자 자신의 멈춤이며, 아무것도 생성되기 전에 멈춘 경우는 정직하게 빈 결과로 타입되지, 만들어진 적 없는 후보장으로 포장되지 않습니다.

### 다른 프로토콜과의 차이

| Protocol | Initiator | Type Signature |
|----------|-----------|---------------|
| Prothesis | AI-guided | `FrameworkAbsent → FramedInquiry` |
| Euporia | Hybrid | `AbstractAporia → ResolvedEndpoint` |
| Proplasma | Hybrid | `DirectionUnrecognizable → DirectionalContrast` |
| **Heuresis** | **User-initiated** | **`CandidateFieldUnderexpanded → DiverseCandidateField`** |

**위치**: Prothesis는 이미 고정된 탐구 위에 분석적 *렌즈*를 놓습니다 — heuresis의 `GenerationFrame`은 전혀 다른 것으로, 병렬 발상 생성을 위한 임시 분할일 뿐 분석적 관점으로 이관되지 않습니다. Proplasma는 후보가 이미 2개 이상 있고 선택 전에 직접 봐야 할 때 미리보기를 제공합니다 — heuresis는 그보다 상류, 후보장 자체를 넓혀야 하는 지점에 있습니다. Euporia는 외재화된 substrate(코드베이스·규칙·과거 세션)에서 결정 좌표를 역추적합니다 — heuresis는 그 substrate를 스캔하지 않으며, 호출 발화와 사용자가 명시적으로 체인한 이전 산출물만 읽습니다.

**라우팅 우선순위**: 후보가 이미 2개 이상 있고 보기만 하면 될 때 → `/preview`; 씨앗 재료가 이 대화 밖에 있을 때 → 수집 프로토콜(예: `/inquire`)을 먼저 돌리고 그 산출을 체인으로 실어옴; 후보장 자체가 얇거나 정체됐을 때 → **`/ideate`**.

## 체인 경계

heuresis는 호출이 실어온 것만 읽습니다: 발화 자체와, 사용자가 명시적으로 지목한 이전 프로토콜의 산출물. 세션 전체·코드베이스·규칙을 스스로 뒤지지 않습니다. 이런 식으로 재료를 체인해오면 그것은 사용자의 의도적인 substrate-우선 선택입니다 — 발화를 쓰기 전에 이미 재료를 읽었으므로, 사전 재료를 보기 *전에* 발상하는 독립-발상의 소유감·다양성 이점은 그 경로에서 성립하지 않습니다. heuresis는 이를 있는 그대로 명시하며, 별도의 우회 장치를 두지 않습니다.

## 사용법

```
/ideate [주제, 또는 이미 가진 몇 가지 조각]
```

heuresis는 호출을 읽고 백지/씨앗 진입을 추론합니다 — 백지라면 아무것도 생성하기 전에 사용자가 고를 추상 프레임 지도를 먼저 엽니다. 이후 매 라운드는 열린 프레임들 위에서 병렬로 후보를 생성하고, 출처를 태깅하며, 아직 열지 않은 프레임을 함께 제시합니다. 프레임을 더 열거나, 하나를 더 깊이 파거나, 직접 이름 붙여 계속할 수 있고, 후보장이 충분히 넓어졌다면 언제든 멈출 수 있습니다. 이렇게 만들어진 후보장은 `/preview`, `/gap`, 혹은 다음 평가 작업으로 흘러갑니다.
