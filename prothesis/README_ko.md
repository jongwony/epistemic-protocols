# Prothesis — /mission (πρόθεσις)

다관점 조사와 실행을 위한 팀 기반 분석 (πρόθεσις: 앞에 놓기)

> [English](./README.md)

## Prothesis란?

그리스어 πρόθεσις(앞에 놓기)의 현대적 재해석으로, **다관점 조사를 위한 팀을 구성하고, 발견 사항에 따라 실행**하는 프로토콜이다.

### 핵심 문제

사용자는 종종 질문에 적합한 분석 프레임워크가 부재한 상태다 (`FrameworkAbsent`). 열린 질문 "어떤 관점을 원하세요?"는 이미 답을 알아야 대답할 수 있다. 관점 선택을 넘어서, 복잡한 질문은 고유한 시점에서의 병렬 조사와 조율된 실행이 필요하다.

### 해법

**Recognition over Recall + 팀 조사**: AI가 관점 옵션을 제시하고, 사용자가 선택하면, 에이전트 팀이 병렬로 조사하고, 발견 사항을 분류하여 동료 검증과 함께 실행한다. 전체 생애주기: 팀 구성 → 병렬 조사 → 교차 대화 → 실행 → 해산.

### 소크라테스 방식과의 차이

| 차원 | 소크라테스 산파술 | Prothesis |
|------|------------------|-----------|
| 지식 원천 | 대화자 내부에 잠재 | 외부에서 제공 |
| 전제 | "당신은 이미 알고 있다" | "당신은 옵션을 모른다" |
| 역할 | 산파 (끌어냄) | 지도 제작자 (경로 공개) |
| 질문 형태 | 열린 질문 (회상 부담) | 선택지 (인지만 필요) |

## 프로토콜 흐름

```
Phase 0: Gather       → 관점 형성을 위한 맥락 수집
Phase 1: Prothesis    → 관점 2-4개 제시 (call AskUserQuestion)
Phase 2: Inquiry      → 선택된 관점별 에이전트 팀 분석 (TeamCreate + teammates)
Phase 3: Synthesis    → 교차 대화 확인 → 수렴점/발산점 정리 → 통합 답변
```

## 사용 시점

**사용**:
- 평가, 비교, 추천 요청
- "전문가 관점에서", "심층 분석" 등 다양한 프레임워크가 가능할 때

**생략**:
- 단순 사실 질문
- 사용자가 이미 관점을 명시했을 때

## 사용법
