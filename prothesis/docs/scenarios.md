# Prothesis 사용 시나리오

> 실사용 전제 — 이 시나리오들은 사용자가 해당 프로토콜을 의식적으로 호출(`/frame`)하는 상황을 기술합니다.
> Skill이 세션에 로드되지 않으면 충돌이 발생하지 않습니다.

**프로토콜**: Prothesis (`/frame`)
**결핍 → 해소**: `FrameworkAbsent → FramedInquiry`

---

### 1. 레거시 모놀리스 마이그레이션 전략 수립 `SW`

**사용자 맥락**: 10년 된 Java 모놀리스를 분해해야 하는 상황. "이 시스템을 현대화해야 한다"고 요청하지만 구체적 접근 방식(Strangler Fig, Big Bang, Domain-Driven 분해 등)을 지정하지 않았다.

**트리거 발화**: "이 시스템을 현대화해야 한다" (목적은 있으나 접근 프레임워크 미지정)

**호출 예시**:
```
/frame 레거시 모놀리스를 현대화해야 하는데, Strangler Fig, Big Bang, DDD 분해 중 어떤 전략이 맞을지 분석해 줘.
```

**결핍(전)**: 목적(현대화)은 명확하나 분석 프레임워크가 부재. Strangler Fig, Big Bang, Domain-Driven 분해 등 다수의 유효한 접근법이 존재하지만 어느 것을 선택할지 기준이 없다.

**해결(후)**: 다관점 병렬 분석으로 각 접근법의 트레이드오프를 분리하고, 수렴점과 발산점을 기반으로 프레임을 고정한다.

**인접 프로토콜 배제**: **Telos** — 목표(현대화)는 이미 정의되어 있으므로 GoalIndeterminate가 아니라, 접근 프레임워크 선택이 필요한 상황이다.

### 2. ESG 공시 전략 수립 `KW`

**사용자 맥락**: 상장사 IR팀장이 2026년 ESG 공시 의무화에 대응해 전략을 수립하려 한다. 목적(규제 대응 + 시장 신뢰)은 명확하나, GRI Standards / TCFD 기후 리스크 / 투자자 관계 / 내부 운영 효율 등 다수의 분석 프레임이 존재한다.

**트리거 발화**: "우리 회사 ESG 전략을 어떻게 잡아야 할까"

**호출 예시**:
```
/frame ESG 공시 의무화 대응 전략을 수립해야 하는데, GRI/TCFD/투자자관계/내부운영 중 어떤 프레임으로 접근해야 할까?
```

**결핍(전)**: 단일 정답 경로가 없고, 어떤 렌즈로 접근하느냐에 따라 공시 우선순위가 완전히 달라지는 상황이다.

**해결(후)**: GRI, TCFD, 투자자 관계, 내부 운영 효율 등 복수 프레임을 병렬 분석해 교집합(우선 공시 항목)과 충돌(단기 비용 vs 장기 가치)을 분리한다.

**인접 프로토콜 배제**: **Telos** — 목표(ESG 공시 대응)는 이미 있고 부족한 것은 분석 프레임워크 선택이다.

### 3. M&A 대상 기업 평가 `KW`

**사용자 맥락**: 사모펀드 투자심사역이 중견 물류회사 인수 검토를 시작하며 평가 방법을 묻는다. DCF 재무 밸류에이션 / 물류 네트워크 전략적 가치 / 규제 리스크 / 기술 인프라 내재가치 등 평가 프레임이 분산되어 있다.

**트리거 발화**: "이 회사를 어떻게 평가해야 하나"

**호출 예시**:
```
/frame 물류회사 인수를 검토 중인데, DCF/전략적 가치/규제 리스크/기술 인프라 중 어떤 평가 프레임이 적절한지 분석해 줘.
```

**결핍(전)**: 목적(인수 의사결정)은 명확하나 어느 각도를 우선시하느냐가 결론을 좌우한다. 재무/전략/규제 축이 교차하여 단일 프레임으로는 불충분하다.

**해결(후)**: DCF, 전략적 가치, 규제 리스크, 기술 인프라 등 복수 프레임을 병렬 적용해 각 관점의 밸류에이션 범위와 교차점을 도출한다.

**인접 프로토콜 배제**: **Syneidesis** — 실행 직전의 갭 점검이 아니라 의사결정 전에 관점 지형 자체를 세팅하는 단계다.

### 4. 팟캐스트 수익화 모델 선택 `CC`

**사용자 맥락**: 주 2회 역사 팟캐스트를 운영하는 크리에이터가 수익화를 시작하려 한다. 스폰서십, 유료 구독(Patreon), 라이브 이벤트, 굿즈, 유료 보너스 에피소드 등 수익 모델이 다양하고, 각 모델이 청취자 경험과 콘텐츠 품질에 미치는 영향이 다르다.

**트리거 발화**: "수익화를 시작하려는데 어떤 모델이 맞을지 분석해 줘"

**호출 예시**:
```
/frame 역사 팟캐스트 수익화를 시작하려는데, 스폰서십/Patreon/라이브이벤트/굿즈를 청취자 LTV와 브랜드 정체성 관점에서 비교 분석해 줘.
```

**결핍(전)**: 어떤 수익 모델을 택할지가 아니라, 어떤 분석 렌즈(청취자 LTV, 제작 비용 구조, 브랜드 정체성 일관성 등)로 비교해야 하는지가 미정인 상태다.

**해결(후)**: 청취자 경험, 수익 구조, 브랜드 정체성 등 복수 프레임을 병렬 분석해 각 수익 모델의 트레이드오프를 구조화한다.

**인접 프로토콜 배제**: **Telos** — "수익화하겠다"는 목표 자체는 확정됨. 접근 프레임워크가 부재한 상황이다.

### 5. 다큐멘터리 내러티브 구조 결정 `CC`

**사용자 맥락**: 독립 다큐멘터리 제작자가 로컬 푸드 운동에 대한 30분짜리 다큐를 제작 중이다. 취재는 완료됐지만 편집 단계에서 연대기적 / 인물 중심 / 문제-해결 / 비교 대조(도시 vs 농촌) 등 복수의 유효한 서사 구조가 있다.

**트리거 발화**: "소재는 다 모았는데 어떤 구조로 편집할지 정해야 해"

**호출 예시**:
```
/frame 로컬 푸드 다큐 편집 구조를 정해야 해. 연대기/인물 중심/문제-해결/비교 대조 중 어떤 서사 구조가 맞을지 분석해 줘.
```

**결핍(전)**: 소재가 같아도 구조에 따라 메시지와 관객 반응이 완전히 달라지므로 각 프레임의 장단점을 병렬 분석해야 한다.

**해결(후)**: 연대기적, 인물 중심, 문제-해결, 비교 대조 등 서사 구조를 병렬 분석해 각 구조의 관객 반응 차이와 메시지 전달 효과를 비교한다.

**인접 프로토콜 배제**: **Hermeneia** — 표현이 애매한 게 아니라 접근 프레임 선택이 필요한 상황이다.
