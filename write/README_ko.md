# Write — /write

세션 인사이트 기반 다관점 블로그 작성

> [English](./README.md)

## Write란?

**세션 인사이트를 체계적이고 출판 가능한 블로그 포스트로 변환하는** 스킬이다. 두 가지 인식론 프로토콜 — 관점 선택을 위한 Prothesis와 갭 검증을 위한 Syneidesis — 을 체계적인 작성 파이프라인에 통합한다.

### 핵심 문제

복잡한 기술 세션을 잘 구조화된 다관점 블로그 포스트로 전환하려면 수동으로 하기 어려운 체계적 분석이 필요하다. 핵심 인사이트가 유실되고, 관점이 탐색되지 않으며, 완전성을 검증하기 어렵다.

### 해결책

**구조화된 작성 파이프라인**: 세션 컨텍스트가 관점 분석, 포맷팅, 작성, 개선, 검증 단계를 거쳐 흐른다. 각 단계에는 명확한 의사결정 포인트와 품질 기준이 있다.

### 프로토콜 통합

| 단계 | 프로토콜 | 목적 |
|------|----------|------|
| Frame | Prothesis (/frame) | 컨텍스트 기반 다관점 렌즈 선택 |
| Validate | Syneidesis (/gap) | 완전성 및 갭 검출 |

## 워크플로우

```
Frame → Format → Draft → Refine → Validate → Finalize
```

| 단계 | 설명 |
|------|------|
| Frame | Prothesis 프로토콜을 통한 분석 관점 선택 |
| Format | 출력 유형(블로그 포스트, 에세이, 뉴스레터, 스레드) 및 언어 선택 |
| Draft | Hook → Context → Framework → Application → Implications 구조의 초안 생성 |
| Refine | 사용자 피드백 기반 반복 개선 루프 |
| Validate | Syneidesis 프로토콜을 통한 갭 검출 |
| Finalize | 최종 편집 적용, 출판 가능 결과물 생성 |

## 콘텐츠 변환

| 세션 요소 | 블로그 요소 |
|-----------|-------------|
| 문제 맥락 | 오프닝 훅 |
| 다관점 분석 | 프레임워크 구조 |
| 수렴점 | 핵심 논점 |
| 발산점 | 토론 섹션 |
| 해결 접근법 | 실행 가능한 방법론 |

## 사용 시기

**사용**:
- 주요 기능 개발이나 조사를 완료한 후
- 프레임워크 개발이나 분석 세션을 블로그 포스트로 변환할 때
- 다관점으로 주제를 분석하여 글을 쓰고 싶을 때

**건너뛰기**:
- 코드 중심 기술 튜토리얼 (일반 작성)
- 문서 업데이트 (직접 편집)
- 검토가 필요 없는 단일 패스 콘텐츠
