# Adapter: claude-session

Progressively disclosed — Read only after the `claude-session` adapter is selected in Phase 0. Satisfies the Vendor Adapter Contract; the core never learns these specifics.

This is a **reflexive, first-party** adapter: unlike an adapter that grounds an *external* vendor's guide for an external tool, `claude-session` grounds Claude's *own* prompting guide to form a handoff initial-prompt for *another Claude session* (a `/remote-spawn` worktree session or a remote-control session). The reference is model-axis organized, so the projection is tailored to the target session's loaded model.

- **Reference**: the Claude prompting best-practices guide. The guide is **model-axis organized** — a general best-practices page plus per-model pages `prompting-claude-{model}` (e.g. `prompting-claude-opus-4-8`, `prompting-claude-fable-5`). "Model-tailored" means branching on the target session's loaded model id to the matching per-model page.
- **capabilities**: the target session's model constraint set, resolved from the model id. **Model resolution is AI-autonomous** (per the `/bound` BoundaryMap for this adapter): default the target model to the *forming session's own loaded model id* (read from the running session, e.g. `claude-opus-4-8`), branch to its `prompting-claude-{model}` page, and apply that model's specific steers (derived from the page per `derive_prompt_schema`, never pinned in this spec). The forming-session model is a relay default; an explicit user-named target model overrides it (the one user-held escape — see boundary overlap). If no per-model page exists for the resolved model, degrade to the general best-practices layer only and flag `model-fallback`.
- **fetch_guide_snapshot**: canonical-external retrieval under the staleness guard, executed as a **progressive-disclosure** sweep (do not bulk-load every model page):
  - **Lv0 — structure**: map the prompt-engineering section to confirm the target model's `prompting-claude-{model}` page exists and reveal the layering (model-specific first → general techniques → migration last). Canonical coordinate: `https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/claude-prompting-best-practices`.
  - **Lv1 — general layer**: extract the model-agnostic best-practices (longform-at-top, document XML tags, query-at-end, communication-style, model self-knowledge).
  - **Lv2 — model layer**: extract the resolved model's page **only** (`…/prompting-claude-{model}`) — never sibling model pages — lifting its full model-specific steer set (thinking/effort, output budget, autonomous-operation/checkpoint, communication/readability, and any reasoning-reproduction sensitivity), not just the intelligence levers. This is progressive disclosure at the data layer, mirroring the skill-layer progressive-disclosure-adapters principle.
  - **staleness**: cross-check against the model's "What's new in Claude {model}" version page and/or the models overview list; these are dated/versioned developer references, so the guard typically passes cleanly. If stale or the model page is unverified, fall back to the general layer (curated seed) and mark the draft `stale-guide`.
- **derive_prompt_schema**: the handoff prompt's slots, **pace-layered**. The adapter carries only the *stable placement principle* (slow layer — model-agnostic, from the general layer); the *model-specific steer content* is **derived dynamically from the resolved model's page** at the density that page prescribes, and is **never pinned here** (per pace layering: the slow-changing placement rule lives in this spec, the fast-changing per-model content is composed from the page at need — never the reverse). A model's specific values baked into this file is the failure mode this structure removes.
  - **Stable placement (slow layer — model-agnostic, always these three in this order):**
    1. **longform context at the top** — the handoff payload (target session's task, project/branch state, constraints, prior decisions) placed *above* the instruction, wrapped in document XML tags (`<context>` / `<documents>` with metadata subtags). *(general: longform-at-top, query-at-end — up to ~30% quality lift)*
    2. **model self-knowledge** — an explicit identity clause for the target model (e.g. "The current model is Claude {name} (`{model-id}`)") when the handoff needs correct self-identification.
    3. **instruction / query at the end** — what the target session is to do, after the longform context.
  - **Model-derived steers (fast layer — lifted from `prompting-claude-{model}` at runtime, in the form and at the density that page prescribes; do NOT carry one model's values into another's handoff):** read the resolved model's page and include only the steers it prescribes. The classes to lift:
    - **thinking / effort** — the model's primary intelligence/latency/cost lever and how to steer it. *(Illustrative contrast, non-normative — always read the resolved page: Opus 4.8 = thinking off unless `thinking:{type:"adaptive"}` set, steerable trigger, effort the primary lever; Fable-class = adaptive-only thinking, no extended-thinking budgets, effort the primary control with `high` the default and lower tiers still strong.)*
    - **output budget** — the model's output-token budgeting guidance for a session spanning subagents/tool calls, exactly as its page states it (never another model's number).
    - **autonomous-operation steers** — **on-point when the target is an unwatched worker** (`/remote-spawn` worktree or remote-control session — it runs with no one watching in real time). Lift the model's checkpoint policy (pause only for genuinely-required input; don't end a turn on a promise/plan), its progress-grounding steer (audit each claim against a tool result before reporting), and any long-run context-budget reassurance its page gives. This is the class the unwatched-worker target most needs and the one the older fixed schema had no slot for.
    - **communication / readability** — the model's user-facing communication steer (e.g. lead-with-the-outcome; re-ground the final summary for a reader who did not watch the run), for a worker that reports back.
    - **option-proposing (optional)** — for design-variety handoffs, the model's design-variety lever (propose N distinct directions before building — the per-model replacement for `temperature`), as its page frames it.
- **project**: render the core `ResolvedIntentIR` (the handoff intent — what the follow-up session should accomplish, plus the carried context/constraints) through the schema — first the stable placement (longform context → self-knowledge → instruction), then the model-derived steers the resolved page prescribes (thinking/effort, output budget, autonomous-operation steers when the target is an unwatched worker, communication/readability). Relay slots (the stable placement order + the model's steers, prescribed by the resolved page) cited; constitution slots (the handoff's task framing, success/verification criteria, scope boundaries) carry a proposed default explicitly flagged. Surface only genuinely necessary missing detail.
- **validate**: check the draft places longform context above the instruction and the instruction at the end; confirm the resolved model matches a known per-model page (else `model-fallback` to the general layer); confirm **every model-derived steer** (thinking/effort, output budget, autonomous-operation, communication) matches the resolved model's page and **not a different model's** — a value carried over from a prior model's page is the pace-layering failure this guards. Surface any slot left underspecified. **Reasoning-echo scan**: if the resolved model's page flags that instructing the model to reproduce, echo, or transcribe its internal reasoning as response text triggers a refusal or silent fallback (the Fable-class `reasoning_extraction` category, which falls back to Opus 4.8), scan the handoff payload — **and any output-style the target session will inherit** (e.g. a reasoning-exposing Ink style) — for reasoning-reproduction instructions; when present, raise `reasoning-echo-unsafe` and recommend the target read its structured `thinking` blocks / use a verbatim send-to-user surface instead of echoing reasoning as response text. **Transport-safety scan**: this adapter's payload routinely carries document XML tags (which a markdown terminal interprets as HTML and truncates unless the payload is displayed verbatim) and shell-active tokens — backticks, `$(`, `${`, `'` — or secret-substitution patterns (e.g. `DD_API_KEY=$(doppler secrets get … --plain)`). When the payload carries any of these, raise `transport-unsafe` and recommend file-based handoff (Write the payload, the carrier reads the file) over an inline shell argument.
- **boundary overlap (`/bound` propagation)**: the **target-model decision** is the bounded escape point. Default disposition is **AI-autonomous** (forming session's loaded model), with an explicit user-named target model as the override. When the target model is genuinely ambiguous — the follow-up session will run an unknown or different model and the user has not named it — propagate to `/bound` for this adapter only, then resume projection with the bounded result.
- **PromptArtifact**: a model-tailored handoff initial-prompt for a follow-up Claude session — carried by `/remote-spawn` into its worktree session, or pasted into a remote-control session. The payload is a **literal opaque artifact**: it routinely contains document XML tags and backticked identifiers, so display it verbatim (fenced) for on-screen reading, and hand it off by **file** — Write the payload, then let the carrier read the file (`/remote-spawn` reads the file, or pass `"$(cat file)"`, whose output is not re-scanned) — rather than as an inline double-quoted shell argument, where backticks / `$(` / `${` undergo command substitution and a literal `$(doppler secrets get …)` would execute and leak a secret. `/forge` emits it and stops; it does not spawn the session, create the worktree, or open branches/PRs (Rule 10, Formation not execution).
