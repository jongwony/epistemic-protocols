# Adapter: claude-session

Progressively disclosed — Read only after the `claude-session` adapter is selected in Phase 0. Satisfies the Vendor Adapter Contract; the core never learns these specifics.

This is a **reflexive, first-party** adapter: unlike `higgsfield` / `gpt-image` / `codex-goals` (which ground an *external* vendor's guide for an external tool), `claude-session` grounds Claude's *own* prompting guide to form a handoff initial-prompt for *another Claude session* (a `/remote-spawn` worktree session or a remote-control session). The reference is model-axis organized, so the projection is tailored to the target session's loaded model.

- **Reference**: the Claude prompting best-practices guide. The guide is **model-axis organized** — a general best-practices page plus per-model pages `prompting-claude-{model}` (e.g. `prompting-claude-opus-4-8`, `prompting-claude-fable-5`). "Model-tailored" means branching on the target session's loaded model id to the matching per-model page.
- **capabilities**: the target session's model constraint set, resolved from the model id. **Model resolution is AI-autonomous** (per the `/bound` BoundaryMap for this adapter): default the target model to the *forming session's own loaded model id* (read from the running session, e.g. `claude-opus-4-8`), branch to its `prompting-claude-{model}` page, and apply that model's specific levers. The forming-session model is a relay default; an explicit user-named target model overrides it (the one user-held escape — see boundary overlap). If no per-model page exists for the resolved model, degrade to the general best-practices layer only and flag `model-fallback`.
- **fetch_guide_snapshot**: canonical-external retrieval under the staleness guard, executed as a **progressive-disclosure** sweep (do not bulk-load every model page):
  - **Lv0 — structure**: map the prompt-engineering section to confirm the target model's `prompting-claude-{model}` page exists and reveal the layering (model-specific first → general techniques → migration last). Canonical coordinate: `https://platform.claude.com/docs/en/build-with-claude/prompt-engineering/claude-prompting-best-practices`.
  - **Lv1 — general layer**: extract the model-agnostic best-practices (longform-at-top, document XML tags, query-at-end, communication-style, model self-knowledge).
  - **Lv2 — model layer**: extract the resolved model's page **only** (`…/prompting-claude-{model}`) — never sibling model pages. This is progressive disclosure at the data layer, mirroring the skill-layer progressive-disclosure-adapters principle.
  - **staleness**: cross-check against the model's "What's new in Claude {model}" version page and/or the models overview list; these are dated/versioned developer references, so the guard typically passes cleanly. If stale or the model page is unverified, fall back to the general layer (curated seed) and mark the draft `stale-guide`.
- **derive_prompt_schema**: the handoff prompt's slots, prescribed by the guide and ordered for the target model:
  1. **longform context at the top** — the handoff payload (target session's task, project/branch state, constraints, prior decisions) placed *above* the instruction, wrapped in document XML tags (`<context>` / `<documents>` with metadata subtags). *(general: longform-at-top, query-at-end — up to ~30% quality lift)*
  2. **model self-knowledge** — an explicit identity clause for the target model (e.g. "The current model is Claude Opus 4.8 (`claude-opus-4-8`)") when the handoff needs correct self-identification.
  3. **instruction / query at the end** — what the target session is to do, after the longform context.
  4. **thinking / effort steer** — model-specific. For Opus 4.8: thinking is off unless `thinking:{type:"adaptive"}` is set, and the trigger is steerable, so include a steer clause when a large handoff would over-trigger thinking ("Thinking adds latency; use it only when it meaningfully improves answer quality — typically multi-step reasoning. When in doubt, respond directly."); treat effort as the primary lever. For other models, derive the equivalent levers from that model's page.
  5. **output budget note** — model-specific (Opus 4.8 at `max`/`xhigh`: start the max output token budget at ~64k so the session has room across subagents/tool calls).
  6. **option-proposing (optional)** — for design-variety handoffs, instruct the target session to propose N distinct directions before building (the per-model design-variety lever that replaces `temperature`).
- **project**: render the core `ResolvedIntentIR` (the handoff intent — what the follow-up session should accomplish, plus the carried context/constraints) through the schema, in slot order (longform context → self-knowledge → instruction → thinking/effort steer → budget). Relay slots (slot ordering, the model's thinking/effort/budget levers — prescribed by the guide) cited; constitution slots (the handoff's task framing, success/verification criteria, scope boundaries) carry a proposed default explicitly flagged. Surface only genuinely necessary missing detail.
- **validate**: check the draft places longform context above the instruction and the instruction at the end; confirm the resolved model matches a known per-model page (else `model-fallback` to the general layer); confirm the thinking/effort and budget guidance match the resolved model's page (not a different model's). Surface any slot left underspecified. **Transport-safety scan**: this adapter's payload routinely carries document XML tags (which a markdown terminal interprets as HTML and truncates unless the payload is displayed verbatim) and shell-active tokens — backticks, `$(`, `${`, `'` — or secret-substitution patterns (e.g. `DD_API_KEY=$(doppler secrets get … --plain)`). When the payload carries any of these, raise `transport-unsafe` and recommend file-based handoff (Write the payload, the carrier reads the file) over an inline shell argument.
- **boundary overlap (`/bound` propagation)**: the **target-model decision** is the bounded escape point. Default disposition is **AI-autonomous** (forming session's loaded model), with an explicit user-named target model as the override. When the target model is genuinely ambiguous — the follow-up session will run an unknown or different model and the user has not named it — propagate to `/bound` for this adapter only, then resume projection with the bounded result.
- **InitialPrompt**: a model-tailored handoff initial-prompt for a follow-up Claude session — carried by `/remote-spawn` into its worktree session, or pasted into a remote-control session. The payload is a **literal opaque artifact**: it routinely contains document XML tags and backticked identifiers, so display it verbatim (fenced) for on-screen reading, and hand it off by **file** — Write the payload, then let the carrier read the file (`/remote-spawn` reads the file, or pass `"$(cat file)"`, whose output is not re-scanned) — rather than as an inline double-quoted shell argument, where backticks / `$(` / `${` undergo command substitution and a literal `$(doppler secrets get …)` would execute and leak a secret. `/forge` emits it and stops; it does not spawn the session, create the worktree, or open branches/PRs (Rule 10, Formation not execution).
